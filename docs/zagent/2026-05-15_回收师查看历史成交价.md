## 一、背景与目标

- **背景**：竞拍结束后回收师需向用户解释报价；当前缺少可直接查阅的**历史成交价**，用户质疑时举证困难。
- **业务目标**：在议价阶段为回收师提供**结构化历史成交信息**，提升解释效率与说服力；不本期承担模型、工作流、生成价等能力。

---

## 二、整体说明

- **适用平台**：现有回收师端订单详情/状态流所在终端（文档以「待查验」等状态页为**同一套 B 端容器**表述，具体 iOS/Android/Web 以项目现状为准）。
- **主要用户角色**：一线回收师。
- **业务流程简述**：用户在多状态（待查验 → … → 鉴定完成）下查看同一商品模块 → 在支持品类下可见「行情信息」「常见瑕疵」→ 进入**历史成交信息**列表页查看/筛选/翻页。

## 三、页面与模块拆分

### 3.1 五状态页 — 商品信息模块（待查验 / 待报价 / 竞拍中 / 竞拍完成 / 鉴定完成）

- **功能概述**：在五类状态页的**商品信息模块**内，**复用同一套**「行情信息」「常见瑕疵」入口、附加信息展开方式、从子页返回后的位置与展开态保持。
- **入口路径**：各状态订单详情 → 商品信息模块（与文档「待查验」规则对齐；无新增独立大块信息区）。
- **关联业务 / 接口**：列表与行情数据由业务接口提供（**接口契约以后端为准**）；前端按「有入口 / 无入口（品类不支持）」展示。

#### 3.1.1 页面结构

- 商品信息模块内需包含：
  - **「行情信息」入口**（准入：当前品类在支持范围内；不支持则不展示入口）。
  - **「常见瑕疵」入口**（同文档；与行情信息并列逻辑由现网一致）。
  - **附加信息**：收纳在模块内，**加号展开/收起**，不单独新开信息区。
  - **「最近竞拍过」标签**（条件满足时展示，见下）。

#### 3.1.2 交互与状态

- **展开/收起**：点击加号展开附加信息；再次点击或点击外侧收起；从子页返回保持展开状态与滚动位置，**不强制回顶**。
- **行情信息 / 常见瑕疵**：点击进入对应详情；返回后回到原位置并保留展开状态。
- **「最近竞拍过」标签**：
  - 展示条件：存在近 **7 天内**符合条件的**历史竞拍单**（具体查询：用户 UID + 关联 UID + 品类必填 + 品牌 + 系列非必填 + 日期近 7 天），并**屏蔽本次回收单**；细则引用文档《上门/门店展示商品历史估价信息》。
  - **7 天是否可配置**：产品待定；前端需兼容**服务端下发是否展示**或**时间窗参数**（待约定）。

#### 3.1.3 视觉与图片资源

- 大神文档中含多状态原型截图（附件名如 `image-2026-4-27_19-49-58.png` 等），需登录大神查看原图；**像素级以设计/大神为准**。

### 3.2 行情信息详情页（新页面 — 文档称「历史成交信息」）

- **功能概述**：展示当前商品在近 **60 天**（默认；**是否可配置待定**）内、已完成**二挂**的成交记录列表；支持分页、成色筛选、下拉刷新、成交价口径说明、商品图预览。
- **入口路径**：上述五状态页商品模块 → 「行情信息」。
- **关联业务 / 接口**：列表、筛选枚举、分页、时间窗、是否走阿波罗 — **以后端/配置约定为准**。

#### 3.2.1 页面结构

- **页面标题**：`历史成交信息`。
- **顶部**：保留**商品信息模块**（图片 + SKU：**有什么展示什么**）。
- **列表区**：
  - 排序：按**成交日期**倒序。
  - 分页：默认每页 **5 条**，可翻页（总页码/加载更多形态以现网组件为准）。
  - 每条至少：**商品图片、SKU 信息（按存在字段展示）、成交价、成色标识**。
- **成色筛选**：
  - 若竞拍环节已有红布林成色：进入页后**默认选中**对应成色筛选项；有成色则展示具体成色标识并参与筛选。
  - 无成色信息：归入「全部」类展示（文档表述）。
- **价格说明**：成交价旁问号 → 展示文案：**成交价 = 竞拍价 + 额外加价（回收师加价 + 加价补贴 + 回收券加价）**。

#### 3.2.2 交互与状态

- **下拉刷新**：刷新列表；刷新后回到**第一页**（与文档一致）。
- **翻页**：常规分页交互。
- **成色筛选**：切换后刷新列表（具体是否重新请求由接口约定）。
- **商品图片**：支持点击**放大预览**；关闭预览后回到**当前列表滚动位置**（不丢位）。
- **空状态**：无成交记录 → 展示 **「暂无信息」**。

#### 3.2.3 准入与 SKU 展示（前端表现）

- 文档以表格说明品类（包袋、手表、首饰等）下必填/非必填 SKU 字段；**前端按接口返回字段渲染**，不臆造字段枚举；字典对齐 **待确认**。

#### 3.2.4 视觉与图片资源

- 行情信息详情线框图：大神附件 `image-2026-5-15_11-19-38.png` 等；请对齐设计。

## 四、非功能需求

- **性能**：列表分页 + 图片懒加载建议与现网订单模块一致；长列表注意避免一次性过大 DOM。
- **兼容性**：与当前回收师 App/WebView 版本策略一致。
- **可访问性 / 其它**：价格口径说明建议支持屏幕阅读器可读（若平台有 a11y 规范则对齐）。
